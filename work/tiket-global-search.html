tiket.com · 2021

Supercharge your search experience with tiket Global Search.

Company tiket.com
Timeline Feb – May 2021 (4 months)
Role Product Designer
Status 🟢 Live Feature
🔍
"Good search doesn't just find things — it helps users discover what they didn't know they were looking for."

Overview

tiket.com offers flights, hotels, trains, buses, activities, and more — but each product category had its own siloed search experience. Users searching for "Bali" would have to navigate to the flights section, then separately to hotels, then to activities. There was no unified way to search across all of tiket.com at once.

Global Search aimed to change that — a single search entry point that surfaced relevant results across all product categories simultaneously.

The Problem

The original search experience had several limitations:

  • No cross-category search — users had to know which product section to navigate to first
  • Search was purely transactional — no inspiration or exploration support
  • No trending or popular destination suggestions
  • The search entry point was buried within each product category

The Diamond Framework

Through keyword analysis and user research, I mapped the search journey into three distinct stages:

  • Starting — The user has an intent but hasn't typed anything. They need inspiration and reassurance that search will work for them.
  • Exploring — The user is typing or has entered a partial query. They need real-time suggestions and autocomplete to refine their intent.
  • Finding — The user has committed to a query and needs comprehensive, well-organized results across all product categories.

The Diamond shape represents how user queries expand from a single intent into broad exploration, then narrow back to a specific result.

Design Highlights

Rotating Search Bar Entry Point

On the homepage, the search bar text rotated through inspirational destination prompts — "Bali beaches", "Tokyo autumn", "Yogyakarta temples" — giving users a gentle nudge toward exploration even before they typed.

Search Landing Page

When a user tapped search without a query, they landed on a rich starting page featuring trending destinations, recent searches, and curated collections. This transformed the "empty state" from dead space into a discovery opportunity.

Cross-Category Results

Search results organized findings by category — flights, hotels, activities — in a single scrollable view, with smart ranking based on the user's search context.

What I Learned

Search design is deceptively complex. What looks like a simple text input contains multitudes of user states, each requiring a different design response. The Diamond Framework helped me and the team articulate these states clearly, which made both the design decisions and the product requirements much easier to align on.

The rotating search placeholder was a small detail that generated outsized impact in user testing — people lit up when they saw destination suggestions they hadn't thought of. Sometimes a micro-interaction is the most powerful discovery tool you have.
